# Security Planning Skill

## Role
Review the design from a security perspective and define security requirements.

## Review Items
1. Authentication/Authorization — Adequacy of access control design
2. Data protection — Handling strategy for sensitive data
3. Input validation — Strategy for processing external inputs
4. Secret management — Storage strategy for keys/tokens
5. Communication security — HTTPS, encryption strategy

## Refer to security-pattern-library.md
- Pre-block design for CRITICAL/HIGH patterns
- Enforce safe implementation patterns

## Output
- List of security requirements
- Threat model (when applicable)
- security-constraints.md draft (when needed)
